### рџЋЇ Logistic Regression вЂ” Titanic Survival Classifier
> *Binary classification. Confusion matrix analysis. Understanding precision vs recall tradeoff.*

![Confusion Matrix](images/day11_confusion_matrix.png)

**What I did:**
- Built a Logistic Regression classifier on Titanic dataset
- Achieved **~82% accuracy** вЂ” correctly predicted 90 non-survivors and 55 survivors
- Analyzed confusion matrix: 19 false negatives (predicted not survived, actually survived) вЂ” in a real scenario, this type of error matters more than false positives
- Visualized survival by gender: females survived at much higher rates (women & children first policy confirmed in data)

![Titanic Gender](images/titanic_survival.png)

**Key learning:** Accuracy alone is misleading. A model predicting "nobody survives" gets 61% accuracy on Titanic вЂ” but it's useless. Confusion matrix tells the real story.

**Code skills demonstrated:** `LogisticRegression`, `confusion_matrix`, `classification_report`, `seaborn.heatmap`

### рџЊі Decision Tree вЂ” Titanic (max_depth=4)
> *Visualized the full decision tree. Understood how the model actually splits data.*

![Decision Tree](images/day12_tree.png)

**What I did:**
- Trained Decision Tree with `max_depth=4` to prevent overfitting
- Visualized the full tree вЂ” first split is on `sex <= 0.5` (encoded: male=1, female=0)
- **The tree confirms what history tells us:** females (sex=False) go right в†’ most survived. Males (sex=True) go left в†’ most didn't survive.
- Analyzed feature importance вЂ” `sex` dominates at 0.58, `pclass` second at 0.20

![Decision Tree Feature Importance](images/day12_importance.png)

**Key learning:** Decision Trees are explainable вЂ” you can trace exactly *why* a prediction was made. This matters in real business applications (banking, healthcare, legal).

**Code skills demonstrated:** `DecisionTreeClassifier`, `plot_tree`, `feature_importances_`, Gini impurity understanding

### рџЊІ Random Forest вЂ” Feature Importance & Model Comparison
> *Ensemble learning. Compared 4 models on same dataset. Understood when Random Forest beats Decision Tree.*

![Random Forest Feature Importance](images/day13_rf_importance.png)

**What I did:**
- Trained Random Forest on Titanic вЂ” **feature importance shifted significantly vs Decision Tree**
- In Decision Tree: `sex` = 0.58 dominant. In Random Forest: `fare`, `age`, `sex` are all ~0.24-0.26 вЂ” more balanced, more realistic
- This shows Random Forest is less biased toward one feature вЂ” it samples features randomly across trees

![Model Comparison](images/day13_model_comparison.png)

**Model Comparison on Titanic Survival:**

| Model | Accuracy |
|-------|----------|
| KNN | ~72% |
| Decision Tree | ~80% |
| Random Forest | ~80% |
| **Logistic Regression** | **~81%** вњ… Best |

**Key learning:** More complex в‰  always better. Logistic Regression outperformed Random Forest here because Titanic survival has strong linear relationships (sex, class). Random Forest wins on complex, non-linear data.

**Code skills demonstrated:** `RandomForestClassifier`, `feature_importances_`, model comparison, choosing the right model
